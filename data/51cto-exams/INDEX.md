# 51CTO 软件设计师历年真题数据集

来源:`https://t.51cto.com/napi/list/detail?id={examId}&need_answer=1`
抓取时间:2026-04-26

## 项目内文件链路

| 文件                               | 作用                                                              |
| ---------------------------------- | ----------------------------------------------------------------- |
| `data/51cto-exams/{examId}.json`   | 原始 API 响应,42 套(本目录)                                       |
| `scripts/split-51cto-exams.py`     | 一次性脚本:从 MCP 工具结果拆分到上述 42 个 JSON                   |
| `scripts/convert-51cto-to-seed.py` | 把原始 JSON 转换为 OSE schema 形式的 seed                         |
| `data/51cto-seed.json`             | OSE 形式的 seed(41 套真题,1579 个 Question 行,96 个 CaseScenario) |
| `src/prisma/seed.ts`               | Prisma 加载脚本（`npm run db:seed` 入口）                         |

## 转换规则要点(详见 `data/51cto-seed.json#metadata`)

- `选择题` → `Question.type=CHOICE`,`session=AM`
- `案例分析` → `Question.type=CASE_ANALYSIS`,`session=PM`,案例小问→`CaseSubQuestion`
- `Question.questionNumber = month*100 + sourceIndex`(避开 5 月/11 月同年同 session 的唯一约束冲突)
- 全部题目挂在占位知识点 `kp-uncategorized` 下,后续可用 AI 重新分类
- `Question.explanation`/`CaseSubQuestion.explanation` 为空字符串(此 API 不返回解析)
- 旧格式案例题(2014-2019)`material_text` 为空,转换为「整段题干作 background + 单一参考答案」
- 新格式案例题(2020+)按 `material_text` 分组,每组一个 CaseScenario,内含多个 sub-question
- 丢弃:`19205 (2025下软设)` 与 `19359 (2025年11月)` 重叠
- 旧案例题(2014-2017)的 `referenceAnswer` 多为「见解析」占位 — 数据源限制

## 原始 JSON 结构(目录内的 `{id}.json`)

每个 `{examId}.json` 是该试卷的完整 API 原始响应,结构:

```
data.data.examine        // 试卷元信息(id/title/total_question/total_score/...)
data.data.question[]     // 题目数组(question_title/option/answer/material_text/show_type_name/...)
data.data.summary        // 各题型小计
data.totalOrder[]        // 题目顺序索引
```

注意:

- `analyze`(解析)字段全部为空,该端点不返回解析。如需解析需另寻接口或练习模式逐题抓取。
- HTML 富文本字段(`question_title`/`option`/`material_text`/`answer`)保留 `<p>`/`<img>` 等标签。
- `material_text` 为案例分析题的题干材料,仅案例题填充。
